# Системный дизайн (Pro)

На экспертном уровне аналитик становится архитектурным фасилитатором: помогает командам договариваться о технических решениях и устраняет риски на уровне системы.

## Основные шаги

1. **Контекст и ограничения** — кто пользуется системой, какие внешние интеграции есть, что необходимо учитывать (SLA, регуляторика, безопасность).  
2. **Выделение доменов** — разделите систему на подсистемы и сервисы, опишите их назначения.  
3. **Потоки данных** — какие данные проходят, где хранятся, какие есть требования к качеству.  
4. **Негативные сценарии** — отказоустойчивость, деградация сервиса, аварийные планы.

## Документы и артефакты

- Контекстная диаграмма (C4 Level 1).  
- Диаграмма контейнеров (C4 Level 2) + описание интерфейсов.  
- Диаграмма последовательностей для сложных взаимодействий.  
- Решённые архитектурные драйверы (ADR).

## Советы

- Фиксируйте компромиссы: почему выбрали данное решение, что будет, если изменить предпосылки.  
- Привлекайте DevOps и безопасность на ранних этапах.  
- Убедитесь, что документация живёт вместе с кодом (репозиторий, Git Hooks, CI).
