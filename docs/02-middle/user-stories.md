# User Stories (Middle)

User story — простой способ описать ценность для пользователя. Формат помогает разработчикам, дизайнерам и аналитикам говорить на одном языке.

## Базовый шаблон

> Как `<роль>` я хочу `<действие>`, чтобы `<результат>`.

Пример:  
> Как менеджер склада я хочу получать уведомление о дефиците товара, чтобы успеть пополнить запас.

## Acceptance criteria

- Описывают границы задачи и условия готовности.  
- Используйте форматы Given/When/Then или чек-листы.  
- Добавьте нефункциональные требования: время реакции, права доступа, устройства.

## Декомпозиция

1. Эпики — крупные потоки ценности.  
2. Capability — группа историй, закрывающих одну бизнес-цель.  
3. User stories — минимальные deliverables, которые можно протестировать.

## Частые ошибки

- История описывает техническое решение, а не пользовательскую потребность.  
- Нет связки с бизнес-метриками.  
- Acceptance criteria не покрывают негативные сценарии.

Хорошая user story запускает обсуждение и находит лучшее решение, а не диктует его.
