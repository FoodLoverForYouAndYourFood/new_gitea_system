# SA Knowledge Hub

Готовая «боевая» библиотека системного аналитика: тёмная Material-тема, мгновенная навигация и кастомные компоненты из коробки. Проект развёрнут на сервере и доступен по внешнему адресу — остаётся наполнять контент.

## Что уже готово
- Настроенная структура уровней: **Новичок → Специалист → Эксперт**.
- Фиолетовая Dark-тема с кастомными стилями и интерактивными блоками.
- Встроенные демо-компоненты (admonition, вкладки, прогресс-бар).
- Скрипт прогресса на JavaScript с сохранением в `localStorage`.
- Автоперезагрузка контента при публикации (всё, что нужно — rebuild на сервере).

## Как пополнять библиотеку
1. **Создайте или обновите статью**  
   Добавьте Markdown-файл в нужный раздел `mkdocs_project/docs/` и оформите ссылки в `mkdocs.yml`, если нужен новый пункт меню.
2. **Закоммитьте и отправьте изменения**  
   ```bash
   git add mkdocs_project/docs
   git commit -m "Добавляет статью <тема>"
   git push
   ```
3. **Обновите продакшн**  
   На сервере (доступ по SSH с ключом):  
   ```bash
   ssh -i id_ruvds root@45.141.102.227
   cd /srv/mkdocs/app
   git pull --ff-only
   /srv/mkdocs/.venv/bin/mkdocs build \
     --config-file mkdocs_project/mkdocs.yml \
     --site-dir /srv/mkdocs/site
   chown -R mkdocs:mkdocs /srv/mkdocs/site
   ```
   Через несколько секунд Nginx отдаёт обновлённую статику.

> Подключение по паролю `Ilovebanana32` всё ещё работает, но рекомендуем использовать SSH-ключ из комплекта проекта (`id_ruvds`), чтобы не делиться секретами.

Готово: база знаний всегда в строю, а авторам достаточно писать тексты и пушить изменения. Если нужна автоматизация (`post-receive` hook или cron), загляните в директорию `scripts/` и адаптируйте под свою инфраструктуру.
