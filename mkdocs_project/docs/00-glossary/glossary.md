# Глоссарий

| Термин | Определение |
| ------ | ----------- |
| **BRD** | Business Requirements Document — документ, описывающий бизнес-потребности и цели проекта. |
| **MRD** | Market Requirements Document — описание рыночных требований и гипотез, которые должен покрыть продукт. |
| **User Story** | Краткое пользовательское требование в формате «кому, что, зачем». |
| **BPMN** | Business Process Model and Notation — нотация моделирования бизнес-процессов. |
| **UML** | Unified Modeling Language — набор диаграмм для описания архитектуры и поведения системы. |
| **XP** | Experience Points — условные «очки опыта», отслеживающие прогресс обучения. |
| **Backlog** | Упорядоченный список задач и требований по продукту. |
| **Scope** | Граница проекта: какие требования включены, а какие оставлены на будущее. |
