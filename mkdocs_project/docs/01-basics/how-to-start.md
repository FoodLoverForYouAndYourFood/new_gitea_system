# Новичок: с чего начать работу над задачей

Пошаговый шаблон, чтобы не потеряться в первых задачах.

## 1. Определи рамки задачи
- Вход: кто попросил, цель, ожидаемый результат, дедлайн.
- Уточни «что будет, если не делать» — это сразу даёт приоритет.
- Зафиксируй метрику успеха и способ измерения.

## 2. Собери факты
- Посмотри данные: логи, SQL/BI, тикеты саппорта, отзывы.
- Набросай પ્રથમ глоссарий: роли, объекты, события.
- Проведи 2–3 коротких интервью (пользователь, бизнес, тех).

## 3. Нарисуй быстрый процесс
- Набросай AS-IS в 5–10 шагах (ручка/доска/Miro).
- Отметь боли и риски в процессе (где теряются данные/время).
- Согласуй схему с участниками, поправь неточности.

## 4. Сформулируй требования
- Истории пользователей (роль → действие → ценность).
- Критерии приёмки (Given/When/Then, позитив + негатив).
- Нефункциональные требования: SLA, производительность, аудит, UX-гайды.

## 5. Приоритизируй
- MoSCoW: Must/Should/Could/Won’t.
- Quick ROI: затраты vs. максимальный эффект.
- Зафиксируй версию MVP (1–2 сценария) и отдельно backlog улучшений.

## 6. Передай в разработку
- Одностраничник/тикет: цель, метрика, сценарий, критерии, ограничения, риски.
- Созвонись с командой: пройтись по документу, ответить на вопросы.
- Добавь ссылку в чат команды, договорись о канале обратной связи.

## 7. Проверь после релиза
- Факт выполнения критериев приёмки (смоук + негатив).
- Договорённая метрика — изменилась ли? Нужны ли корректировки.
- Собери обратную связь от пользователей и саппорта.

## Быстрый чек-лист
- [ ] Поняты цель и ценность.
- [ ] Есть краткий процесс AS-IS с болями.
- [ ] Истории и критерии приёмки описаны.
- [ ] НФТ перечислены.
- [ ] MVP выделен, backlog зафиксирован.
- [ ] Договорённость с командой и канал поддержки после релиза.
